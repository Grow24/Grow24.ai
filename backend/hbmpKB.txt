2. Non-Negotiable Rules and Conventions (PBMP Doctrine)
2.1 Management meaning (canonical)
Management = Goals → Strategy → Plan → Execute (Projects) → Operate, with Change Management pervasive.
Avoid naming like "Planning & Management" (planning is a subset of management).

2.2 Naming rules (strict)
Use noun-based names for solutions/processes/artifacts (e.g., "Customer Segmentation", "Market Sizing", "Pipeline Governance").
Avoid "/" in names. If needed, split into parent-child (e.g., "Industry Analysis" → child "Competitive Analysis").
Prefer single responsibility naming: one concept per node.

2.3 Strategy lifecycle (canonical sequence)
Strategic Analysis → Strategy Formulation (includes Strategic Choices) → Strategy Implementation & Control.
Do not use "Strategic Choices" as a separate stage; it is part of Formulation.

2.4 Content transformation rule (book screenshots)
Near-verbatim paraphrase, sentence-by-sentence, preserving headings/sub-headings, ordering, and meaning.
Use different wording.
If a page is hard to read, intelligent guesses may be used and marked as assumptions.

2.5 Wide table mandate (minimum fields)
Questions (down to lowest level; include 5W1H).
Techniques + algorithms/activities (BABOK tools mapped where relevant).
Data needed + data sources (external + internal; anomalies as hints).
How data combines (triangulation, joins, cohorts, segmentation).
Visualizations (dashboards + layouts; which chart answers which question).
Output artifacts + embedded outputs (metrics/insights inside artifacts).
Response/actions (what decision or corrective action follows).

2.6 Hyper-local and anomaly lens
External scanning includes microeconomic and hyperlocal signals (city/region/cluster; where relevant even apartment-level).
Questions should support anomaly detection: cumulative vs non-cumulative, WoW/MoM/QoQ/YoY, FYTD vs last year, seasonality, cohort comparisons, leading vs lagging indicators.

4. Unified Measurement and Cascade Model (GOST + OGSM + OKR/KPI)
4.1 Definitions (strict)
Goal (G): SMART (metric + baseline + target + time + scope).
Objective (O): measurable sub-outcome that causally drives the goal.
Strategy (S): coherent choices (where-to-play/how-to-win; capability bets; trade-offs).
Tactics (T): concrete initiatives/actions/projects.

4.2 Cascade rule (key)
G/O/S/T are relative to hierarchy level. A Strategy at Board level can become a Goal or Objective at CXO/Department level.

4.3 Cascade levels required
Board → CXO → Department → Individual

7. Website Bot Prompt Contract (Anti-hallucination)
7.1 Grounding rules
State product facts only if supported by retrieved KB items with audience=Website, visibility=Public, status=GA, confidence=Confirmed.
If insufficient evidence: ask up to 3 clarifying questions OR offer safe CTAs (demo/contact/brochure) OR say you do not have that information yet.
Do not invent, do not guess, do not fill missing data.
Major claims must include provenance (KB item IDs).

7.2 Intent processing
Produce canonical Formal English intent for every message.
Map to an Intent ID; if uncertain, offer top 2-3 options and ask user to choose.
Produce intent JSON per schema; if required slots missing, ask for them; if JSON cannot be produced, ask user to re-prompt.

7.3 Retrieval policy
Retrieve per intent retrieval_targets; filter to Website/Public/GA by default.
Retrieve multiple types when relevant: Solution + FAQ + Evidence; optionally Process/Template public descriptions.
If factual query and Evidence hits < 1: abstain or ask clarifying questions.

8. Unified Measurement Framework
PBMP uses a unified GOST + OGSM + OKR/KPI framework that cascades from Board level to Individual contributors.

8.1 Core Principles
- Goals are SMART: Specific, Measurable, Achievable, Relevant, Time-bound
- Objectives are measurable sub-outcomes that causally drive goals
- Strategies are coherent choices (where-to-play, how-to-win, capability bets, trade-offs)
- Tactics are concrete initiatives, actions, and projects

8.2 Cascade Hierarchy
Board Level → CXO Level → Department Level → Individual Level
Each level's Strategy can become a Goal or Objective at the next level down.

8.3 Management Lifecycle
Goals → Objectives → Strategy & Tactics → Plan → Execute (Projects) → Operate (Operations)
Change Management overlays all stages.
